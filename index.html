Algorithm Details

Encryption Process

For each character m in plaintext:
1. Convert to numeric value
2. Generate random k where 1 < k < p-1
3. Compute a = gk mod p
4. Compute b = m × (gm)k mod p
5. Output pair (a, b)

Decryption Process

For each ciphertext pair (a, b):
1. For each possible m in alphabet:
2. Compute am mod p
3. Check if b ≡ m × am mod p
4. When match found, convert m to character
Mathematical Foundation: The security relies on the difficulty of the discrete logarithm problem. For each plaintext character, multiple ciphertext pairs are possible due to the random exponent k, providing homophonic properties.
