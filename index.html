🧠 Algorithm Explanation

Encryption Process

For each byte value in the input:

  1. Apply custom mapping using the mapping key
  2. Treat mapped byte as numerical value m (0-255)
  3. Generate a cryptographically secure random number k
  4. Compute a = gk mod p
  5. Compute b = m × gm×k mod p
  6. The ciphertext is the pair (a, b)
Ciphertext = (gk mod p, m × gm×k mod p)

Decryption Process

For each ciphertext pair (a, b):

  1. For each possible byte value m (0-255):
  2. Compute am mod p
  3. Check if b ≡ m × am mod p
  4. When match is found, reverse the custom mapping
  5. Output the original byte value
For each m in 0-255: check if b ≡ m × am mod p

The decryption requires trying all possible m values (0-255), which is efficient.

Why this works: am ≡ (gk)m ≡ gk×m mod p, and b = m × gm×k mod p, so b ≡ m × am mod p
Important: To decrypt, you need the same Prime (p), Generator (g), and Mapping Key values used for encryption.
